# i18n 国际化设计文档

## 概述

为铭润科技全站（公开站点 + Console 后台 + 认证页面）添加国际化支持。默认语言中文，支持英文，架构预留多语言扩展能力。同步将全站品牌从 QIHANG 替换为铭润科技 / Mingrun Tech。

## 技术选型

**next-intl** — 专为 Next.js App Router 设计的 i18n 方案。

选择理由：
- 原生支持 App Router 的 `[locale]` 动态路由段
- 内置 middleware 处理语言检测、重定向、cookie 持久化
- `localePrefix: 'as-needed'` 支持默认语言无 URL 前缀
- Server Component 和 Client Component 均可使用
- TypeScript 类型安全
- 社区活跃，文档完善

## URL 结构

采用"默认语言无前缀"方案：

| 场景 | URL |
|------|-----|
| 中文首页 | `mingrun-tech.com/` |
| 中文产品页 | `mingrun-tech.com/product` |
| 英文产品页 | `mingrun-tech.com/en/product` |
| 中文 Console | `mingrun-tech.com/app` |
| 英文 Console | `mingrun-tech.com/en/app` |
| 中文登录 | `mingrun-tech.com/login` |
| 英文登录 | `mingrun-tech.com/en/login` |

中文用户的 URL 与现有结构完全一致，零迁移成本。

## 翻译文件结构

按命名空间拆分，每种语言一个文件夹：

```
messages/
  zh/
    common.json           ← 导航、Footer、语言切换器、品牌、通用 UI
    home.json             ← 首页
    product.json          ← 产品页
    ecosystem.json        ← AI 生态页
    demo.json             ← Demo 页
    pricing.json          ← 定价页
    support.json          ← 支持页
    updates.json          ← 动态页
    auth.json             ← 登录、注册页
    error.json            ← 404、错误页
    console.json          ← Console 通用（侧栏、顶栏、状态栏、命令面板）
    console-projects.json
    console-datasets.json
    console-train.json
    console-models.json
    console-eval.json
    console-settings.json
    console-devices.json
    console-billing.json
  en/
    (同结构)
```

Key 采用扁平点号命名：

```json
{
  "nav.product": "产品",
  "nav.ecosystem": "AI 生态",
  "hero.title": "看见周围，\n理解周围。"
}
```

新增语言只需：
1. `routing.ts` 的 `locales` 数组加一项
2. `messages/` 下新建文件夹，复制 key 结构，填入翻译

**命名空间加载策略：** `i18n/request.ts` 在每次请求时合并加载当前语言的所有命名空间到一个 `messages` 对象。文件按命名空间拆分是为了方便维护和查找，不是为了按需加载——项目规模下全量加载开销可忽略。`getTranslations('product')` 直接从预加载的 `messages` 中取对应命名空间的 key。

## 路由结构改造

现有结构：
```
app/
  layout.tsx              ← 根布局 (html, body, fonts)
  not-found.tsx
  (public)/
    layout.tsx, page.tsx, product/page.tsx, ...
  (console)/
    layout.tsx, app/page.tsx, ...
  login/page.tsx
  register/page.tsx
```

改造后：
```
app/
  layout.tsx              ← 精简根布局 (透传 children，不渲染 html/body)
  api/                    ← API 路由，不进入 [locale] 段
  [locale]/
    layout.tsx            ← 实际 HTML 根 (html lang, body, fonts, NextIntlClientProvider)
    not-found.tsx
    (public)/
      layout.tsx, page.tsx, product/page.tsx, ecosystem/page.tsx,
      demo/page.tsx, pricing/page.tsx, support/page.tsx, updates/page.tsx
    (console)/
      layout.tsx, app/page.tsx, app/projects/page.tsx,
      app/projects/[id]/page.tsx,
      app/datasets/page.tsx, app/datasets/[id]/page.tsx,
      app/train/page.tsx, app/train/[id]/page.tsx,
      app/models/page.tsx, app/models/[id]/page.tsx,
      app/eval/page.tsx, app/settings/page.tsx,
      app/devices/page.tsx, app/billing/page.tsx
    (auth)/
      login/page.tsx, register/page.tsx
```

### 根布局与 locale 布局分工

**`app/layout.tsx`（根布局）：** 精简为透传容器，不渲染 `<html>` 或 `<body>`——仅 `return children`。

**`app/[locale]/layout.tsx`（locale 布局）：** 成为实际的 HTML 根，接收 `params.locale`，职责包括：
- 渲染 `<html lang={locale}>` 和 `<body>`
- 加载字体（Inter、JetBrains Mono）
- 包裹 `NextIntlClientProvider`，向 Client Components 提供翻译上下文
- 现有 `Providers`（QueryClientProvider、ViewTransition）集成到此层

**`app/api/`：** 保持在顶层，不进入 `[locale]/` 段。middleware matcher 已排除 `api` 路径。

### generateStaticParams

所有 `[locale]` 下的页面需要导出 `generateStaticParams` 以支持静态生成：

```ts
export function generateStaticParams() {
  return [{ locale: 'zh' }, { locale: 'en' }];
}
```

## next-intl 基础设施

### i18n/routing.ts

```ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['zh', 'en'],
  defaultLocale: 'zh',
  localePrefix: 'as-needed'
});
```

### i18n/request.ts

```ts
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;
  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  // 合并加载所有命名空间
  const namespaces = [
    'common', 'home', 'product', 'ecosystem', 'demo', 'pricing',
    'support', 'updates', 'auth', 'error', 'console',
    'console-projects', 'console-datasets', 'console-train',
    'console-models', 'console-eval', 'console-settings',
    'console-devices', 'console-billing',
  ];
  const messages: Record<string, any> = {};
  for (const ns of namespaces) {
    messages[ns] = (await import(`../../messages/${locale}/${ns}.json`)).default;
  }

  return { locale, messages };
});
```

### i18n/navigation.ts

```ts
import { createNavigation } from 'next-intl/navigation';
import { routing } from './routing';

export const { Link, redirect, usePathname, useRouter } = createNavigation(routing);
```

所有 locale 感知的链接和导航使用 `i18n/navigation.ts` 导出的 `Link` 和 `useRouter`，替换 `next/link` 和 `next/navigation`。

### middleware.ts

```ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';

export default createMiddleware(routing);

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)']
};
```

### next.config.js

用 `createNextIntlPlugin` 包裹现有配置：

```js
const createNextIntlPlugin = require('next-intl/plugin');
const withNextIntl = createNextIntlPlugin();

const nextConfig = {
  // 现有配置...
  async redirects() {
    return [
      // 重定向保持不变，middleware 会在重定向前注入 locale
      { source: '/how-it-works', destination: '/product', permanent: true },
      { source: '/docs', destination: '/support', permanent: true },
      { source: '/contact', destination: '/support#contact', permanent: true },
    ];
  },
};

module.exports = withNextIntl(nextConfig);
```

语言检测优先级：
1. URL 路径（`/en/product` → 英文）
2. Cookie（用户上次选择）
3. `Accept-Language` 请求头
4. 回退到 `zh`

## 组件集成模式

**Server Components（页面、布局）：**
```tsx
import { getTranslations } from 'next-intl/server';

export default async function ProductPage() {
  const t = await getTranslations('product');
  return <h1>{t('hero.title')}</h1>;
}

export async function generateMetadata() {
  const t = await getTranslations('product');
  return { title: t('meta.title') };
}
```

**Client Components（交互组件）：**
```tsx
'use client';
import { useTranslations } from 'next-intl';

export function LanguageSwitcher() {
  const t = useTranslations('common');
  // ...
}
```

Client Components 使用 `useTranslations()` 需要 `NextIntlClientProvider` 包裹。该 Provider 放在 `app/[locale]/layout.tsx` 中，与现有 Providers 组合：

```tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

export default async function LocaleLayout({ children, params }) {
  const { locale } = await params;
  const messages = await getMessages();
  return (
    <NextIntlClientProvider messages={messages}>
      <Providers>{children}</Providers>
    </NextIntlClientProvider>
  );
}
```

### Link 组件迁移

现有 `PublicDocumentLink` 组件包裹 `next/link`，在公开站和 Console 中广泛使用。改造方案：

- `PublicDocumentLink` 内部的 `next/link` 替换为 `i18n/navigation.ts` 导出的 `Link`
- 该 `Link` 自动处理 locale 前缀（中文无前缀，英文加 `/en/`）
- 组件接口不变，调用方无需修改

## 内容迁移策略

现有文案散落在三处：
1. 组件内硬编码（`SiteHeader.tsx` 的 `NAV_ITEMS` 等）
2. 内容文件（`public-story-content.ts`、`home-content.ts`）
3. 页面内联（各 `page.tsx` 的标题、描述）

### 迁移方式

**简单文案**（导航标签、按钮、标题）：直接迁移到 JSON，组件用 `t('key')`。

**结构化内容**（`public-story-content.ts` 中的 scenes）：
- 可翻译的文本（title、summary、details 的 label/value）迁移到 JSON
- 不可翻译的数据（`viewerPatch` 对象、`icon` 路径、`color` 值）保留在 TypeScript 文件中作为纯数据
- 组件从两个来源组合：翻译文本来自 `t()`，结构数据来自 TS 常量

示例：
```ts
// lib/product-data.ts (不可翻译的结构数据)
export const PRODUCT_SCENES = [
  { id: 'surface', viewerPatch: { ... }, icon: '...' },
  { id: 'structure', viewerPatch: { ... }, icon: '...' },
];
```
```json
// messages/zh/product.json (可翻译文本)
{
  "scenes.surface.title": "表面工艺",
  "scenes.surface.summary": "...",
  "scenes.structure.title": "内部结构",
  "scenes.structure.summary": "..."
}
```

迁移完成后删除 `public-story-content.ts` 和 `home-content.ts`。

## 语言切换器

**位置：** 公开站 SiteHeader 右侧（"开始使用"按钮左边），Console TopBar 同位置。

**交互：**
- 当前两种语言：点击直接切换（`中` ↔ `EN`），无需下拉菜单
- 未来语言增多后改为下拉选择器
- 切换后跳转到当前页面的对应语言版本
- Cookie 记住用户选择

**实现：** 使用 `i18n/navigation.ts` 的 `useRouter` 和 `usePathname` 切换 locale。

**视觉：**
- 文字按钮，无边框，与导航项风格一致
- 公开站和 Console 分别跟随各自主题色

## SEO

- 每个页面的 `<head>` 中输出 `<link rel="alternate" hrefLang="zh" href="..." />` 和 `hrefLang="en"`
- `generateMetadata` 中设置 `alternates.languages`
- `og:locale` 标签跟随当前语言

## 翻译质量要求

- 所有翻译必须准确、地道、自然，无 AI 翻译痕迹
- 中文为基准，英文翻译需符合英语母语表达习惯
- 技术术语保持行业通用译法
- 品牌名（铭润科技 / Mingrun Tech）不翻译，按语言版本直接使用对应形式
