# 铭润控制台重设计 — 设计规格

**日期**: 2026-03-15
**状态**: 已批准
**范围**: 控制台前端全面重设计（后端 API、数据库、基础设施不变）

---

## 1. 背景与动机

当前控制台以 ML 工程师工作流为核心设计（数据集 → 标注 → 训练 → 模型注册 → 评估），对铭润的目标用户来说过于底层。铭润的产品定位是 **开源模型 + 用户 DIY 微调 + 耳机交互 = 私人 AI 助手平台**，控制台应当反映这一定位。

### 当前问题

1. **视觉偏"工具化"** — 深色主题 + 系统字体 + 蓝色强调，像开发者工具面板
2. **布局模板化** — 每个页面同一模式（eyebrow + 标题 + 表单 + DataTable），缺乏层次
3. **导航过于简陋** — 48px 图标条无文字，层级太扁平
4. **数据展示缺深度** — 基础表格，无排序/筛选/分页，复杂数据无合适可视化
5. **术语面向开发者** — "数据集""模型版本""别名"等概念对普通用户不友好
6. **字体无品牌感** — 系统默认字体，缺乏辨识度

### 目标

- 将控制台从"ML 工程平台"转变为 **"AI 助手工作室"**
- 实现 **渐进式披露**：上限高（极客可调所有参数），下限低（新手三步上手）
- 建立温暖高级的品牌视觉风格

---

## 2. 用户分层

| 用户类型 | 典型模型选择 | 使用深度 | 界面体验 |
|---------|------------|---------|---------|
| 年轻消费者/学生 | 轻量模型 (7B) | 选模型 + 简单配置 | 向导流程，开箱即用 |
| 职场专业人士 | 中等模型 (V3) | 上传专业资料 + 定制人格 | 画布工作台，按需深入 |
| 科技极客 | 大型模型 (72B) | 精调参数 + 版本管理 + 评估 | 开发者视图，完整控制 |

用户分层不通过"选择模式"来实现，而是通过 **模型选择自然分流** + **渐进式展开**。

## 3. 视觉风格

### 3.1 设计气质

创意工作室 + 高端科技感。像 Notion 一样好上手，像 Apple 一样有质感。温暖的配色和圆润的界面传递"人情味 AI"的品牌理念。

### 3.2 色彩系统：暖奶油白

| 色彩角色 | 用途 | 色值 | CSS 变量 |
|---------|------|------|---------|
| 底色 Base | 页面大面积背景 | `#f5f0eb` | `--bg-base` |
| 表面 Surface | 侧边栏、顶栏 | `#faf7f4` | `--bg-surface` |
| 卡片 Card | 内容卡片 | `#ffffff` | `--bg-card` |
| 悬浮 Raised | 下拉菜单、弹窗 | `#ffffff` | `--bg-raised` |
| 强调 Accent | 主按钮、活跃态 | `#c8734a` | `--accent` |
| 强调渐变 | 品牌元素 | `#c8734a → #e8925a` | `--accent-gradient` |
| 主文字 | 标题、正文 | `#2a2018` | `--text-primary` |
| 次文字 | 描述、辅助 | `#9a8a7a` | `--text-secondary` |
| 淡文字 | 占位、禁用 | `#b0a090` | `--text-muted` |
| 边框 | 卡片、分割 | `#e8e0d8` | `--border` |
| 边框淡 | 次级分割 | `#ede6de` | `--border-light` |
| 成功 | 运行中、已完成 | `#2a8a5a` | `--success` |
| 警告 | 训练中、待处理 | `#d4923a` | `--warning` |
| 错误 | 失败、删除 | `#c44a4a` | `--error` |
| 强调背景 | 活跃态底色 | `rgba(200,115,74,0.1)` | `--accent-soft` |
| 成功背景 | 成功态底色 | `rgba(42,138,90,0.08)` | `--success-soft` |
| 错误背景 | 错误态底色 | `rgba(196,74,74,0.08)` | `--error-soft` |

### 3.3 排版

- **字体**: 英文标题使用 DM Sans，正文/中文使用 PingFang SC / Noto Sans SC。DM Sans 几何感强、现代感好，与暖色调配色形成对比张力。通过 `next/font/google` 加载（Next.js 在构建时下载字体并自托管，不依赖 Google CDN 运行时加载，避免 FOUT）。
- **字号层级**:
  - 页面标题: 18px, weight 700
  - 面板标题: 14px, weight 600
  - 正文: 13.5px, weight 400
  - 辅助/标签: 11px, weight 600, uppercase, letter-spacing 0.8-1.2px
  - 小字: 10.5px
- **行高**: 正文 1.7, 标题 1.3
- **字间距**: 标题 -0.3px, 标签 +0.8-1.2px

### 3.4 圆角

- 卡片/面板: 14px
- 按钮: 9px
- 标签/Badge: 6px
- 头像: 50%
- 输入框: 10px

### 3.5 阴影

极淡暖色阴影，轻盈有层次:
- 卡片: `0 2px 8px rgba(42,32,24,0.04)`
- 悬浮: `0 4px 16px rgba(42,32,24,0.08)`
- 弹窗: `0 8px 32px rgba(42,32,24,0.12)`

### 3.6 动效

克制的过渡动画:
- 页面切换: 150ms ease-out, opacity + translateY(8px)
- 面板展开/收起: 200ms ease-in-out
- 按钮悬浮: 150ms, translateY(-1px) + 阴影加深
- 状态变化: 300ms 渐变过渡
- 列表栏收起: 200ms width 过渡

### 3.7 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| Desktop | ≥1024px | 完整三栏：图标栏 + 列表栏 + 主内容 |
| Tablet | 768-1023px | 列表栏默认收起，图标栏保留，点击图标展开列表栏为浮层。画布工作台 2×2 网格保持，但区块内容紧凑化。 |
| Mobile | <768px | 图标栏变为底部 Tab Bar（5 项：AI、知识、训练、对话、更多），列表栏和顶栏隐藏，"更多"展开设备/设置。画布工作台 2×2 网格变为单列堆叠。 |

移动端底部 Tab Bar 复用现有 `MobileMenuProvider` 上下文，替换当前的 `UnifiedMobileNav` 实现。

## 5. 核心流程：创建 AI 助手

### 5.1 首次创建：分步向导

```
步骤 1: 选择基座模型 → 步骤 2: 添加知识 → 步骤 3: 设定人格 → 步骤 4: 命名并完成
```

#### 步骤 1: 选择基座模型

- 展示 3 档模型卡片（轻/中/强），每张包含:
  - 模型名称（如 Qwen3.5-7B）
  - 一句话定位（如"响应极快 · 日常对话 · 简单任务"）
  - 推荐标签（如"推荐入门""最受欢迎""高级用户"）
- 选中态: 2px 强调色边框 + 轻微阴影
- 可展开查看模型详细参数（高级用户）

#### 步骤 2: 添加知识（可跳过）

- 拖拽上传区域，支持 PDF、TXT、DOCX、MD
- 上传后显示文件列表 + 索引进度
- "跳过，稍后添加"按钮

#### 步骤 3: 设定人格（可跳过）

- 预设人格模板网格（专业顾问、学习伙伴、生活助手、创意搭档等）
- 选择模板后可编辑描述文字
- 风格标签选择（专业/友善/幽默/严谨/...）
- "跳过，使用默认"按钮

#### 步骤 4: 命名并完成

- AI 助手名称输入
- 头像/图标选择（预设图标 + 颜色）
- 概览确认（模型、知识、人格的摘要）
- "开始训练"主按钮

完成后自动跳转到画布工作台，训练在后台进行。

### 5.2 日常编辑：画布工作台

路由: `/app/assistants/[id]`

四个区块卡片在一个页面上，2×2 网格布局:

```
┌────────────────────┬────────────────────┐
│   基座模型          │   知识库            │
│   当前模型 + 更换   │   资料列表 + 上传    │
│                    │                    │
├────────────────────┼────────────────────┤
│   人格设定          │   技能              │
│   描述 + 标签编辑   │   技能列表 + 添加    │
│                    │                    │
└────────────────────┴────────────────────┘
```

- 每个区块右上角有"编辑"/"管理"入口
- 点击"编辑"在区块内展开编辑状态，或打开侧面板
- 顶部操作栏: AI 名称 + 状态 + "试用对话"按钮 + "保存并训练"按钮
- 训练状态实时显示在顶部（进度条 + 百分比）

#### 高级展开

每个区块支持"展开高级选项":

| 区块 | 默认视图 | 高级展开 |
|------|---------|---------|
| 基座模型 | 模型名称 + 一句话描述 | 模型参数详情、推理配置 |
| 知识库 | 文件列表 + 状态 | 数据项详情、标注编辑、版本快照管理 |
| 人格设定 | 描述文字 + 风格标签 | 完整 system prompt 编辑器 |
| 技能 | 技能名称列表 | 技能参数配置、函数定义 |

#### 开发者视图

在设置中开启"开发者模式"后，画布上额外显示:
- 每个区块显示底层 ID（project_id, dataset_id, model_id 等）
- 训练区块显示完整参数 JSON、日志流、指标曲线
- 模型别名管理（prod/staging/dev）
- 原始 API 响应查看

## 7. 新增功能

### 7.1 对话调试

- 路由: `/app/chat`
- 选择 AI 助手 + 版本，在浏览器内直接对话
- **v1 实现**: 纯前端 mock UI。对话界面展示完整的聊天交互框架（消息列表、输入框、AI 助手选择器），但 AI 回复为前端 mock 响应（固定的示例回复 + 打字机动效）。消息状态存在组件 state 中，不持久化。这与现有训练/推理的 mock 策略一致。
- **后续迭代**: 需要后端新增 `/api/v1/chat` 端点支持真实推理、多轮对话、SSE 流式响应、对话历史持久化。此后端工作不在本次前端重设计范围内。
- 可切换不同 AI 助手快速对比效果

### 7.2 人格模板库

- 向导步骤 3 中的预设模板
- 内置模板（6-8 个常用场景）
- 模板包含: 人格描述 + 风格标签 + 推荐技能
- 选择后可自由修改

### 7.3 开发者视图切换

- 设置页面中的开关
- 开启后控制台全局显示额外的技术细节
- 不改变布局，只在现有区块中追加信息
- 状态持久化到用户偏好

### 7.4 技能的后端映射

"技能"在后端对应 `training_jobs.params_json` 中的 tools/functions 字段。v1 中，技能配置在创建训练任务时传入（复用现有 `POST /api/v1/train/jobs` 端点的 `params_json` 字段）。画布工作台中的"技能"区块为前端状态管理——用户编辑技能列表，点击"保存并训练"时将技能配置合并到 `params_json` 中一并提交。不需要新的数据库表或 API 端点。

## 8. 不变的部分

以下系统层面完全不动，前端重设计只涉及展示层:

- **后端 API** — 所有 `/api/v1/` endpoint 保持不变
- **数据库 schema** — PostgreSQL 表结构不变
- **S3 上传流程** — presign + direct upload 不变
- **认证系统** — JWT + HttpOnly Cookie 不变
- **国际化架构** — next-intl 保持，需新增/修改 message keys
- **Docker 基础设施** — docker-compose 配置不变
- **Celery 任务系统** — 后台任务处理不变

## 9. 技术实现约束

- **框架**: Next.js 16 App Router + TypeScript（保持现有）
- **样式**: TailwindCSS + CSS 变量（替换现有 globals.css 中的控制台样式）
- **组件库**: shadcn/ui（保持，按需扩展）
- **状态管理**: TanStack Query（保持）
- **动画**: Framer Motion（保持）
- **图表**: recharts（保持，可能扩展）
- **路由变更**: 旧路由需要重定向到新路由

## 10. 成功标准

1. 新用户从注册到创建第一个 AI 助手 < 3 分钟
2. 所有现有 API 功能在新界面中可达
3. 开发者视图暴露与旧控制台同等的技术细节
4. 中英文完整支持
5. 移动端基本可用（响应式布局）
