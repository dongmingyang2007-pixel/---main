# 控制台消费者化重建设计

**日期：** 2026-03-19
**类型：** 重建式改造（方案 B）
**目标用户：** 普通耳机用户（非技术背景）

## 概述

将控制台从"AI 开发平台"重新定位为"AI 助手管理中心"。删除所有面向开发者的技术模块（数据集、训练、模型注册、评估），以新的产品定位为起点重新设计所有页面，旧代码只在组件级别复用。

## 核心设计原则

1. **记忆是统一抽象** — 助手的知识、技能、用户了解全部表达为"记忆"，而非数据集、知识库、训练结果等技术概念
2. **模型是可替换引擎** — 记忆系统作为持久层与模型解耦，用户换模型不丢失上下文
3. **零技术概念暴露** — 用消费者语言替代所有技术术语
4. **全站使用原创 SVG 图标** — 不使用 emoji

## 删除的模块

以下页面及其相关组件、路由、导航项全部移除：

- `/app/knowledge` — 数据集/知识库管理
- `/app/training` — 训练任务
- `/app/models` — 模型注册/版本管理（被"发现"页替代）
- `/app/assistants/[id]/versions` — 助手版本历史
- Canvas Workbench — 卡片网格式的助手详情页

## 新导航结构

从 7 项（助手、知识、训练、聊天、模型、设备、设置）改为 5 项：

| 导航项 | 路由 | 说明 |
|---|---|---|
| 首页 | `/app` | Dashboard，进入控制台的第一眼 |
| 对话 | `/app/chat` | 聊天记录、语音通话 |
| 记忆 | `/app/memory` | 记忆图谱、列表、导入导出 |
| 设备 | `/app/devices` | 耳机连接与管理 |
| 发现 | `/app/discover` | 记忆包社区 + 模型市场 |

设置通过导航栏底部的用户头像入口访问，不占主导航位。
助手详情通过 Dashboard 助手卡片点击进入（`/app/assistants/[id]`），不作为独立导航项。

## 布局系统

### 可展开侧边栏

取消现有的 IconBar（52px 纯图标）+ ListPanel（180px 可折叠列表）双栏结构，改为单一可展开侧边栏：

- **默认状态**：56px 紧凑图标栏
- **悬停状态**：平滑展开至 200px，显示文字标签和用户信息
- **动画**：使用 spring easing（`cubic-bezier(0.22, 1, 0.36, 1)`），文字标签有滑入效果（`translateX(-8px) → 0`，延迟 100ms）
- **底部**：用户头像 + 名字 + 设置入口
- **Active 状态**：锈色左边框指示条 + 背景色块

### 桌面端

侧边栏（56px/200px）+ 主内容区（剩余空间）。取消 PanelLayout 的三栏 resizable 结构。

### 移动端

底部标签栏（MobileTabBar），调整为新的 5 项导航。

## 页面设计

### 1. Dashboard 首页（`/app`）

新增页面，替代当前的助手列表页跳转。

**内容：**
- 欢迎语 + 助手学习动态（"你的助手今天学习了 3 条新记忆"）
- 助手卡片：头像、名字、在线状态、记忆数、"开始聊天"CTA。点击进入助手档案页
- 快速统计：本周对话数、记忆总数、已连接设备数
- 最近对话列表：标题 + 时间，点击跳转到对话详情

### 2. 助手档案页（`/app/assistants/[id]`）

从 Canvas Workbench 卡片网格重构为 Profile 页面风格。

**头部：**
- 大头像（88px，渐变背景，可编辑）
- 助手名字 + 性格标语
- 元数据行：创建日期、对话总数、记忆总数
- CTA："开始聊天"（主按钮）、"设置"

**标签页：**
- **概览**（默认）：四张卡片网格 — 性格摘要、活跃度统计、模型配置、知识文件
- **性格**：性格模板展示 + 自定义描述编辑
- **知识**：已上传文件列表 + 上传新文件
- **模型**：四个能力槽位的配置（见模型能力系统）

**模型配置四槽位：**

| 槽位 | 用户理解 | 对应技术模态 |
|---|---|---|
| 对话 | 助手的大脑 | 文本 LLM |
| 视觉理解 | 助手的眼睛 | 图像/视频模型 |
| 语音识别 | 助手的耳朵 | ASR |
| 语音合成 | 助手的嘴巴 | TTS |

- 全模态模型：选一个，四槽位自动填满
- 单模态/部分模态模型：用户逐个槽位选择组装
- 每个槽位显示当前模型名 + "更换"链接（跳转到发现页模型区）

### 3. 创建向导

从 4 步变为 3 步，核心变化是性格优先、隐藏模型选择。

**旧流程：** 选模型 → 上传知识 → 设性格 → 命名完成
**新流程：**

1. **选择性格**
   - 6 个预设模板卡片（温暖陪伴、高效助理、学习导师、创意伙伴、冷静顾问、幽默玩伴）
   - 每个模板：自定义图标 + 名称 + 一句话描述
   - "或者"分隔符下方：自由文本输入框（自定义性格描述）
   - 系统根据性格模板自动匹配最佳模型

2. **命名 + 形象**
   - 左侧：头像（可点击更换）+ 颜色选择器（6 色）
   - 右侧：名字输入 + 开场白输入
   - 创造情感连接

3. **教它知识**（可选）
   - 顶部提示："这一步是可选的，你可以随时在助手设置中添加知识"+ "跳过，直接完成 →"
   - 拖拽上传区域
   - 支持 PDF、Word、TXT、Markdown

**导航：** 底部"上一步 / 进度点 / 下一步"，进度点动画指示当前步骤。

### 4. 对话页面（`/app/chat`）

保留当前的对话列表 + 聊天区双栏布局，增强体验。

**对话列表（左栏 260px）：**
- 顶部：标题 + 新建对话按钮
- 搜索框
- 按日期分组的对话列表（今天 / 昨天 / 本周 / 更早），比纯时间戳更易扫读
- 每条显示：标题 + 预览文字 + 时间

**聊天区（右栏）：**
- 顶部栏：助手头像 + 名字 + 在线状态 + 语音通话按钮 + 更多选项
- 消息区：
  - 用户消息：锈色气泡，靠右
  - 助手消息：浅色卡片（`#faf7f4` 底 + 边框），靠左，带小头像
  - **记忆高亮**：助手回复下方出现标签"记住了：xxx"，点击可跳转到记忆图谱对应节点。这让用户直观感知助手在学习。
- 输入区：
  - 圆角输入框，多行自适应高度
  - 输入框下方左侧：能力芯片行
    - "+"按钮：展开更多工具
    - **搜索**芯片：联网搜索，默认自动（AI 判断是否需要），可手动开关
    - **深度思考**芯片：深度推理模式，默认自动（AI 根据问题复杂度判断），可手动开关
    - 芯片状态：自动（默认，无高亮）/ 开启（高亮）/ 关闭（划线）
  - 输入框下方右侧：附件上传、语音输入、发送按钮
  - 发送按钮为圆形锈色实心
  - **自适应逻辑**：默认所有工具为 auto 模式，AI 自行判断是否调用。用户可以手动切换为强制开启或强制关闭。语音对话模式下无 UI 操控，完全由 AI 自主判断

### 5. 记忆页面（`/app/memory`）

从助手详情子页面提升为顶级模块。

**默认视图：力导向图谱**（图谱优先，列表为切换选项）

**顶部栏：**
- 标题 + 记忆总数
- 图谱/列表视图切换
- 操作按钮：导出、导入、新建记忆

**图谱视图：**
- 中心节点：助手（大节点，渐变背景，呼吸动画）
- 记忆节点三色系统：
  - 锈色（`#c8734a`）— 永久记忆（用户明确确认的事实）
  - 蓝色（`#4a8ac8`）— 学习到的记忆（从对话中提取）
  - 紫色（`#8a5ac8`）— 记忆包（从社区下载的）
- 记忆包以集群形式呈现：一个大节点（包名）周围带小卫星节点（包内记忆）
- 连线表示关联关系，粗细/颜色区分关联强度
- 交互：悬停显示标签，点击右侧滑出详情面板（毛玻璃背景 `backdrop-filter: blur(12px)`）
- 浮动 UI：左上搜索框、右上过滤胶囊、右下缩放控件

**详情滑出面板（300px）：**
- 关闭按钮
- 类型标签 + 分类
- 记忆标题 + 创建时间
- 记忆内容（可编辑）
- 来源（从哪次对话提取的）
- 关联记忆标签
- 操作：编辑、删除

**列表视图：**
- 左侧列表面板（320px）：搜索 + 分类过滤标签（全部/个人/知识/偏好/记忆包）+ 记忆条目
- 右侧详情面板：与图谱点击后的详情相同

**导入导出：**
- 导出：下载全部记忆为文件（JSON 格式），或选择性导出某个分类/记忆包
- 导入：上传记忆文件，作为可编辑副本导入（fork 模式，非只读引用）
- 从对话中提取的新记忆可自然归入已有的记忆包分类中

**设备 Hero 卡片：**

**设置区域（两栏网格）：**

### 7. 发现页面（`/app/discover`）

替代现有模型市场页面，整合记忆包社区和模型市场。

**顶部栏：** 标题 + 搜索框

**分类标签页：** 全部 / 记忆包 / 模型

**记忆包区域：**
- 卡片网格（3 列）
- 卡片内容：图标 + 名称 + 作者 + 描述 + 下载量 + 记忆条数 + "获取"按钮
- 热门/新上架角标
- 最后一个卡片位是**上传 CTA**："分享你的记忆包"— 鼓励用户成为创作者，帮助冷启动

**记忆包详情页（`/app/discover/packs/[id]`）：**
- 包名、作者、描述、记忆条数、下载量
- 记忆内容预览列表
- "获取"按钮 → 作为可编辑副本导入到用户的记忆库
- 下载后用户可自由编辑、删除、补充

**模型区域：**
- 卡片网格（3 列）
- 卡片内容：Logo + 名称 + 提供商 + 能力标签 + 消费者语言描述 + 操作按钮
- 能力标签示例："对话"、"视觉"、"深度推理"、"数学"、"编程"
- 当前使用的模型标注"当前使用"高亮标签
- 操作按钮文案："替换对话模型"/ "替换视觉模型"（明确告诉用户这个操作的含义）
- 全模态模型标注"全能"标签，点击后一键替换所有槽位

**模型详情页（`/app/discover/models/[id]`）：**

参照阿里云百炼的详细程度，但使用消费者语言呈现：

- **头部**：模型名称 + 提供商 Logo + 能力标签（文本生成、视觉理解、深度思考等）
- **模型简介**：消费者语言的描述，解释这个模型擅长什么，适合什么场景
- **能力矩阵**：
  - 输入模态图标：文本、图像、音频、视频（用图标标识支持/不支持）
  - 输出模态图标：文本、图像、音频（同上）
  - 核心能力：联网搜索、工具调用、深度思考、缓存加速等（勾选列表，消费者化命名）
- **适用场景**：2-3 个典型使用场景描述（如"日常对话"、"文档分析"、"代码辅助"）
- **操作按钮**：根据模型覆盖的能力槽位显示对应的替换按钮
- 不暴露：token 数量、API 价格、batch 推理等技术细节

**模型描述语言规范：**
- 用消费者语言：
  - "能看懂图片和文档" 而非 "多模态视觉语言模型"
  - "擅长复杂推理和数学" 而非 "Chain-of-Thought reasoning with 64K context"
  - "说话更自然" 而非 "低延迟流式 TTS"
  - "支持联网搜索" 而非 "function calling + web search tool"
  - "能深度思考复杂问题" 而非 "Chain-of-Thought reasoning"

## 模型能力系统

### 四个能力槽位

每个助手有四个能力槽位，对应设备的四个通道：

| 能力 | 用户理解 | 技术模态 | 示例模型 |
|---|---|---|---|
| 对话 | 大脑 — 负责思考和回复 | 文本 LLM | Qwen 3.5 Plus, DeepSeek R1 |
| 视觉理解 | 眼睛 — 看懂拍到的东西 | 图像+视频 | Qwen VL Max |
| 语音识别 | 耳朵 — 听懂说的话 | ASR | Whisper Large |
| 语音合成 | 嘴巴 — 决定说话声音 | TTS | CosyVoice |

### 模型选择逻辑

- **全模态模型**：覆盖多个能力，选择后自动填充对应槽位。如果覆盖全部四个则一键完成
- **单模态模型**：只覆盖一个能力，用户逐槽位选择组装
- **创建向导中**：系统根据性格模板自动匹配推荐的模型组合，用户无需手动选择
- **后续更换**：通过助手档案页的模型标签页或发现页进行

### 记忆保障模型可替换性

由于采用独立的记忆体系（非 fine-tuning），模型替换不影响助手的知识和性格：
- 记忆存储在向量数据库中，不绑定任何模型
- 性格通过 system prompt + 记忆注入实现，与底层模型无关
- 换模型 = 换引擎，助手的"灵魂"（记忆）不变

## 自适应工具与思考系统

### 核心理念

工具调用和深度思考由 AI 自主判断，不需要用户主动管理。类似 ChatGPT 的 auto 模式。

### 工具类型

| 工具 | 用户看到的名称 | 触发场景 |
|---|---|---|
| 联网搜索 | 搜索 | 用户询问实时信息（天气、新闻、股价等） |
| 深度思考 | 深度思考 | 复杂的推理、数学、分析类问题 |
| 图像识别 | （自动，无 UI） | 用户通过摄像头拍摄内容 |

### 文字对话中的 UI

输入框下方显示能力芯片，用户可以覆盖 AI 的自动判断：

- **auto（默认）**：芯片无高亮，AI 自行决定是否调用
- **强制开启**：芯片高亮（锈色），本次对话强制使用该工具
- **强制关闭**：芯片显示划线，本次对话禁止使用该工具
- 点击芯片循环切换三个状态
- "+"按钮可展开查看所有可用工具

### 语音对话中的行为

语音模式下没有 UI 操控入口，AI 完全自主判断：
- 用户说"今天北京天气怎么样" → AI 自动调用搜索
- 用户说"帮我分析一下这个方程" → AI 自动启用深度思考
- 用户闲聊 → 不调用任何工具，快速回复

### 助手回复中的工具指示

当 AI 调用了工具时，在回复中显示指示（与记忆高亮类似的小标签）：
- "已搜索：北京天气"
- "已深度思考"
- 让用户了解助手做了什么，但不需要用户主动操作

## 记忆包系统

### 核心概念

记忆包（Memory Pack）是一组预构建的记忆节点，可以被打包、分享、下载。它统一了传统意义上的"技能"、"插件"、"知识库"概念。

### 规则

1. **下载即拥有**：下载的记忆包是可编辑的副本（fork 模式），不是只读引用
2. **可自由修改**：用户可以编辑、删除、补充包内的记忆
3. **自然归入**：对话中提取的新记忆可以归入已下载的记忆包分类
4. **可上传分享**：用户可以将自己的记忆按分类打包上传到社区

### 图谱中的呈现

记忆包在力导向图谱中以集群形式呈现：
- 一个较大的紫色节点代表包本身（显示包名）
- 周围环绕较小的紫色节点代表包内的各条记忆
- 与其他记忆的关联通过连线表示

## 视觉设计

### 色彩体系

保留并沿用现有的暖色调控制台主题（`data-theme="console"`）：

- 背景：`#f5f0eb`（奶油底色）/ `#faf7f4`（卡片/面板）
- 侧边栏：`#ede6df`
- 强调色：`#c8734a`（锈色，用于 CTA、active 状态、用户消息气泡）
- 文字：`#2a2018`（主文字）/ `#9a8a7a`（辅助文字）/ `#6a5a4a`（二级文字）
- 边框：`#e8e0d8`
- 成功/在线：`#2a8a5a`

### 记忆三色系统

- 永久记忆：`#c8734a`（锈色）
- 学习记忆：`#4a8ac8`（蓝色）
- 记忆包：`#8a5ac8`（紫色）

### 圆角

- 卡片：`16-20px`
- 按钮/输入框：`10-12px`（常规）/ `20-24px`（pill 形 CTA）
- 面板/侧边栏：`16px`（外层 shell）

### 图标

全站使用 SVG 线条图标（stroke-based），不使用 emoji。图标风格：
- 线宽：2px
- 端点：`stroke-linecap: round`
- 连接：`stroke-linejoin: round`
- 默认颜色跟随文字颜色，active 状态使用强调色

### 动画

- 侧边栏展开：spring easing `cubic-bezier(0.22, 1, 0.36, 1)`，duration 300ms
- 文字标签滑入：translateX(-8px) → 0，delay 100ms
- 中心节点呼吸：box-shadow 脉冲，3s ease-in-out infinite
- 设备连接灯：绿色 box-shadow 脉冲，2s ease-in-out infinite
- 页面切换：保留现有 View Transitions API 支持
- 卡片 hover：translateY(-2px) + shadow 增强

### 排版

沿用现有字体配置：
- 标题：DM Sans
- 正文：Inter
- 代码：JetBrains Mono
