# Foundation Design System Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish the shared design foundation (tokens, fonts, libraries, shared components) that both the Public Site and Console upgrade tracks depend on.

**Architecture:** New design tokens (CSS custom properties) are added alongside legacy tokens in `globals.css` — no legacy removal yet. Tailwind config maps new token-based colors. shadcn/ui is initialized with 7 selective components, themed via a token bridge. GSAP + ScrollTrigger, Framer Motion, and react-resizable-panels are installed. Fonts switch from DM Sans + Sora to Inter + JetBrains Mono.

**Tech Stack:** Next.js 16, Tailwind CSS 3.4, shadcn/ui (cmdk, Radix), GSAP + ScrollTrigger, Framer Motion, react-resizable-panels, Inter, JetBrains Mono

**Spec:** `docs/superpowers/specs/2026-03-13-full-site-ui-upgrade-design.md`

### Task 9: Console dark theme integration

**Files:**
- Modify: `apps/web/app/(console)/layout.tsx`

- [ ] **Step 1: Add theme attributes to console layout**

The console layout needs to apply `data-theme="console"` for custom tokens and `class="dark"` for shadcn. Since Next.js App Router doesn't allow route-group layouts to modify `<html>`, use a client-side effect.

Replace the entire file with:

```tsx
"use client";

import { useEffect } from "react";
import { ProjectProvider } from "@/lib/ProjectContext";
import { AppShell } from "@/components/AppShell";

export default function ConsoleLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  useEffect(() => {
    const root = document.documentElement;
    root.setAttribute("data-theme", "console");
    root.classList.add("dark");
    return () => {
      root.removeAttribute("data-theme");
      root.classList.remove("dark");
    };
  }, []);

  return (
    <ProjectProvider>
      <AppShell title="Control Console">{children}</AppShell>
    </ProjectProvider>
  );
}
```

Key points:
- `"use client"` is required for `useEffect`
- On mount: adds `data-theme="console"` + `class="dark"` to `<html>`
- On unmount (navigating away from console): removes both attributes
- This ensures shadcn components auto-switch to dark theme in console
